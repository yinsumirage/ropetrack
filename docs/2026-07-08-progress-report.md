## 0. 任务设定与评价口径（术语先行，后文直接引用）

### 0.1 任务与"绳长"信号

单目 RGB 手姿态估计（输出 MANO 参数化手模型的 21 个关节 3D 位置）在**手部被遮挡时显著退化**。
我们的设想：在手套/腕带形态的设备上布 5 根从各指尖到手腕锚点的"绳"，实时读出 5 个长度标量；
用这 5 个标量在**测试时**修正视觉模型的预测，不改动、不重训视觉模型本身。

**必须先行声明的定性（贯穿全文）**：当前阶段的绳长读数是**由 GT 关节位置派生的模拟传感器**。
它的现实性边界由第 3 节的噪声/掉线消融给出；"5 个标量远少于 63 维姿态答案、不构成平凡泄漏"
由第 5 节的 oracle 差距证明。凡对外表述，此定性必须出现。（硬件原型现已制成，但本报告所有
数字仍来自模拟读数；真实读数的标定与验证是第 7 节阶段 1–2 的内容。）

### 0.2 指标与术语

**PA-MPJPE（mm，全文主指标）**：预测与 GT 的 21 关节平均位置误差，计算前先做 Procrustes
对齐（求最优旋转+平移+缩放后再比较）。它度量"姿态形状"错误——手指到底弯成什么样——而不含全局
位置/朝向/尺度误差。选它的理由：遮挡损伤恰恰集中在"手指弯曲状态估错"这类形状错误上，PA 口径
把问题聚焦在这里。数值越低越好。

其余术语集中列出，后文不再重复解释：

| 术语 | 含义 |
|---|---|
| 增益 / delta（mm） | 修复后 PA 减修复前 PA，**负 = 改善**。基线与修复后走**同一个** MANO 解码管线（"同解码器"口径），避免协议差异污染差值 |
| occluded-tip / clean-tip 切片 | 只在"被遮挡指尖 / 未遮挡指尖"子集上统计的增益。遮挡指尖是损伤集中地，也是绳信号应当起作用的地方 |
| 闭合率（closure） | 优化（或学生）把"预测绳长 vs 传感器读数"之间残差消除掉的比例。注意：闭合率只衡量"服从传感器"的程度，**不等于姿态变好**（见 2.3 的"有害闭合"证据） |
| 动作空间 mult5 / flex15 | 修正量 α 的参数化：mult5 = 每指 1 个弯曲乘子（5 标量）；flex15 = 每指 3 个关节各一个弯曲增量（15 标量）。α 经 tanh 限幅，学习式变体最后一层零初始化 |
| 门控 gate010 | 硬规则（非学习）：绳残差 < 0.1（归一化单位）的手指不做任何修正（α 置 0）。gated fraction = 残差超过阈值、被允许修正的 valid 手指占比 |
| hard split | 程序化生成的遮挡评测集：mask70 = 手部区域约 70% 被遮挡块覆盖；finger_end80 = 每根手指远端 80% 被遮挡（五指全部受累）。程序化生成保证与 clean 版本严格同图成对，可做受控对照 |
| backend | 被修复的视觉模型：WiLoR（主 backend）与 HaMeR（第二 backend，用于验证结论不是 WiLoR 特有）。全部在 GT 检测框下评测，排除手部检测器差异 |
| 数据集 | FreiHAND（训练劣化 split 32,560 样本 / 评测劣化 split 3,960 样本）；HO3D v2（跨数据集评测）与 HO3D v3（训练数据来源），HO3D 为手—物交互视频 |

**决策纪律**（贯穿各阶段）：每个实验在开跑前注册可证伪的预测与数值门槛，跑完对照门槛裁决；
负结果记录在案而不丢弃。文中每个"转向"处会给出当时的判断依据。

## 1. 第一幕：遮挡打垮 RGB baseline（FreiHAND +5.07 mm，HO3D +1.90 mm）

| 数据集（WiLoR，GT bbox） | clean PA（mm） | mask70 hard PA（mm） | 遮挡损伤 |
|---|---:|---:|---:|
| FreiHAND | 4.99 | 10.07 | **+5.07（误差翻倍）** |
| HO3D v2 | 7.54 | 9.44 | +1.90 |

读法：同一个模型、同一批底图，只是加上程序化遮挡。FreiHAND 上误差直接**翻倍**（4.99 →
10.07），这确立了问题规模，也定义了后文一切"修复"的分母——汇报修复效果时统一说"修复了
5.07 mm 损伤中的多少"。HO3D 的**边际**损伤明显更小（+1.90）：HO3D 是真实手持物交互视频，
它的 clean 基线 7.54 里**已经含着真实物体遮挡的代价**——合成遮挡叠上去时，一部分"遮挡税"
已经付过了。这个差别会贯穿全文：HO3D 上的可修复量与天花板（第 5 节）都系统性小于 FreiHAND，
原因在此，不是方法在 HO3D 上失灵。

**失败模式诊断**（绳信号选型的依据）：对逐样本误差做诊断分析发现，损伤集中表现为**被遮挡的蜷曲
手指被预测得过于张开**——视觉证据缺失时，模型回退到"手指伸直"的先验。这直接指出了对症的修正
方向：手指的**弯曲程度**。而"指尖—手腕距离"（绳长）恰好是弯曲程度的单调观测量——手指越蜷，
指尖离手腕越近。这就是为什么第二幕的修正动作空间选"弯曲标量"（mult5/flex15），而不是任意的
姿态修正。

**设计说明（为什么用程序化 hard split 而不是真实遮挡数据）**：真实遮挡数据没有"同图去遮挡"的
成对参照，无法把"遮挡损伤"从数据分布差异中剥离出来；程序化遮挡给出严格成对的 clean/hard 对照，
且严重度可控（mask70 与 finger_end80 是两类结构不同的遮挡：面积型 vs 指端型）。真实遮挡场景由
HO3D（手持物体的真实交互视频）作为跨数据集验证补上。

## 2. 第二幕：绳长测试时优化，修复约 1/3 损伤

方法：对每个测试样本，冻结视觉模型，只对 α（5 或 15 个弯曲标量）做梯度优化，目标是让修正后
姿态的**预测绳长逼近传感器读数**；α 应用后走同一 MANO 解码得到修正姿态。无任何训练、无任何
标注参与——这是纯测试时（test-time）修复。

整条管线如下（示意；模块只插在"基线姿态之后、MANO 解码之前"，视觉模型完全不动）：

```text
RGB 图像 ──► [ 冻结视觉模型 WiLoR / HaMeR ] ──► 基线姿态 θ̂ ─────────────┐
                                                                        ▼
绳传感器 ──► 读数 r (5 维) ──► [ 修正模块（本项目新增）→ α ] ──► θ̂ ⊕ α ──► 同一 MANO 解码
                                    │                                    │
                                    │                                    ▼
                                    │                              修正后 21 关节
                                    ├─ 教师：对 α 做 400 步梯度下降，min ‖r̂(θ̂⊕α) − r‖²
                                    ├─ 学生：一次前向 MLP(θ̂, r, 绳残差, 有效位) → α
                                    └─ 门控 gate010：绳残差 < 0.1 的手指强制 α = 0（硬规则）

r̂(·) = 从当前修正姿态解码出的预测绳长；⊕ = 沿弯曲方向叠加 α（tanh 限幅 ≤0.5）
```

> **【插图位 0：pipeline 框图】** 正式版建议按上方示意作图：视觉模型画为灰色"冻结"块，
> 修正模块高亮为新增部分，教师/学生作为模块内两种可互换的实现，门控画在模块出口。

**为什么把学习问题先退化成逐样本优化**：我们最早的学习式尝试（直接回归 45 维姿态修正的 MLP）
在训练缓存上拟合但在 held-out 评测上变差——教训是**输出空间超出了 5 个标量所能决定的范围**
（"可观测性原则"：学习模块只应输出绳能决定的量）。逐样本优化 + 低维弯曲动作空间是这一教训的
直接产物，后来它反过来成为第四幕蒸馏的教师。

### 2.1 剂量—响应诊断：小增益是"优化不足"，不是"信号没用"

**转向的理由**：P0 阶段默认优化配方下绳修复的增益很小（全关节仅 0.16 mm）。此时有两个竞争假设：
(a) 绳信号本身信息量不足；(b) 优化强度不够、残差根本没被消掉。区分它们的诊断证据是残差闭合率
——默认配方只闭合了 3.8% 的绳残差，即传感器的话**根本没被听进去**，指向 (b)。于是固定动作空间
（mult5，P0 默认）不变，只扫优化强度（步数 × 学习率），看剂量—响应关系：

| 配方 | 绳残差闭合率 | 全关节增益（mm） | 修复后 PA（mm） |
|---|---:|---:|---:|
| —（基线，不修正） | 0% | – | 10.07 |
| lr=2, steps=120（P0 默认） | 3.8% | 0.16 | 9.91 |
| lr=8, steps=400 | 27.8% | 1.00 | 9.07 |
| lr=32, steps=120 | 30.1% | 1.07 | 9.00 |
| lr=32, steps=400 | 41.8% | 1.40 | 8.67 |

（本表"增益"列为改善幅度，正 = 改善；后文各表统一用带符号差值，负 = 改善。"修复后 PA" =
基线 10.07 − 增益，两个端点与运行记录直接核对一致：默认配方 9.91、最强配方 8.67。）

读法：闭合率从 3.8% 提到 41.8%，增益同步单调地从 0.16 mm 涨到 1.40 mm。如果假设 (a) 成立，
加大剂量后增益应当很快平台化；实际是干净的单调剂量—响应关系——**信号是真的，此前只是没用上**。
这一步是整个项目的关键转折：它把"绳信号只值 0.16 mm"的错误结论翻转为后面所有结果的基础。

### 2.2 动作空间与门控怎么选出来的（栅格对比），以及冠军配方

修正量 α 的参数化（动作空间）不是直接钦点的，有三个候选，各自动机明确：

- **mult5**（每指 1 个弯曲**乘子**）：最早的探针空间。缺陷后来暴露：乘法修正在基线姿态接近
  "伸直"（近零）时乘什么都是零——存在**死区**，而失败模式恰恰是"该弯的手指被预测成伸直"。
- **flex5**（每指 1 个**加性**弯曲标量，沿该指的耦合梯度方向）：加性消灭死区；且容量与信号
  匹配——绳每指只给 1 个约束，就只给每指 1 个自由度，指内无冗余。
- **flex15**（每关节 1 个加性弯曲标量，每指 3 个）：容量过剩版本，冗余靠正则与限幅压制。

**门控 gate010**（在此第一次实际使用，复述机制）：修正前先看每根手指的绳残差——基线预测的
绳长与传感器读数已经一致（残差 < 0.1）的手指，说明视觉对它大概率没错，强制 α = 0 完全不动；
只修残差大的手指。它是硬规则，不参与学习。

赛前注册的预测是 **flex5 会赢**（容量匹配论：15 维空间由 5 个读数驱动，自由度过剩应当有害）。
强配方下的完整栅格结果（FreiHAND mask70；finger_end80 上同一栅格给出相同排序，不重复列）：

| 动作空间 × 门控 | 闭合率 | 修复后 PA（mm） | 全关节增益（mm） | 遮挡指尖增益（mm） |
|---|---:|---:|---:|---:|
| **flex15 + 门控** | 48.5% | **8.39** | **-1.68** | **-5.47** |
| mult5 + 门控 | 40.2% | 8.52 | -1.55 | -4.71 |
| flex5 + 门控 | 40.7% | 8.65 | -1.42 | -4.49 |
| flex15 无门控 | 49.7% | 8.67 | -1.39 | -4.59 |
| mult5 无门控 | 41.8% | 8.67 | -1.40 | -4.09 |
| flex5 无门控 | 43.7% | 8.88 | -1.19 | -3.84 |

读法，三个结论：① **门控对三个动作空间全部是正收益**（每个空间带门控的行都优于其无门控行）；
② 冠军是 flex15 + 门控——预注册的 flex5 预测**被数据反驳**：在 L2 正则、tanh 限幅、零初始化
梯度下降的最小范数偏置、残差门控四重约束下，容量过剩不是坏事，反而给了优化器"把弯曲分配到
正确关节"的余地（这次反驳记录在案——选型不是事后挑好看的数字）；③ 闭合率最高的格（flex15
无门控，49.7%）PA 反而不是最优——这个反直觉现象单独展开在 2.3。

冠军配方（后文一切"教师"均指它）：

```
rope + flex15 + gate010 + steps=400 / lr=32 / alpha_l2=0.001
```

（复现口径说明：优化损失为 batch 均值，故有效学习率与 batch 大小耦合；所有公开结果固定
batch=512。）

冠军配方在五个评测设置上的成绩（**HO3D 与 HaMeR 均为零重调直接套用**，配方从未接触这些设置）：

| 评测设置 | PA（mm） | 全关节增益 | 遮挡指尖增益 | 闭合率 |
|---|---:|---:|---:|---:|
| FreiHAND mask70（WiLoR） | 8.39 | **-1.68** | **-5.47** | 48.5% |
| FreiHAND finger_end80（WiLoR） | 8.32 | -1.66 | -3.45 | 50.6% |
| FreiHAND mask70（HaMeR） | 9.12 | -1.70 | -5.64 | 51.3% |
| HO3D v2 mask70（WiLoR，零重调） | 8.39 | -1.05 | -2.09 | 54.2% |
| HO3D v2 mask70（HaMeR，零重调） | 8.67 | -0.66 | -0.82 | 55.4% |

读法（按行）：主设置 mask70 上全关节 -1.68 mm——占 5.07 mm 遮挡损伤的约 **33%**，且修复集中在
该集中的地方：遮挡指尖 -5.47 mm。第 2 行换遮挡类型、第 3 行换 backend，增益几乎不变（-1.66 /
-1.70），说明修复机制不挑扰动形态、不是 WiLoR 特有。第 4–5 行换数据集（真实手物交互视频）且
**完全不重调超参**，增益变小但方向一致——一部分原因第一幕已给出（HO3D 的边际损伤本来就小），
跨数据集仍是最弱的轴，这个观察贯穿后文（第四幕的多教师改进与第 7 节的 DexYCB 扩展都由它
驱动）。

**一句话总结第二幕**：一个 400 步的测试时优化 + 5 个标量的传感器读数，免训练地修复了约 1/3 的
遮挡损伤。

### 2.3 门控的机制证据："有害闭合"真实存在

为什么必须有门控、且门控是硬规则？证据就在 2.2 的栅格表里，对比 flex15 的两行：**不带门控**
闭合了**更多**绳残差（49.7% vs 48.5%），修复后 PA 却**更差**（8.67 vs 8.39，增益 -1.39 vs
-1.68 mm）。也就是说，对绳残差本来就小的手指（视觉预测已经和传感器一致）强行把残差压到零，
会把正确的姿态拉坏——闭合率和姿态质量在这里**解耦**，我们称之为"有害闭合"。门控用"放弃一
部分闭合"换取"不伤害无辜手指"。这也是全文把闭合率只当诊断量、从不当优化目标汇报的原因（同一
机制在第 3 节噪声大时以"冤枉干净手指"的形式再次出现）。

## 3. 第三幕：传感器现实性——噪声与掉线

**为什么必须做这一节**：绳读数由 GT 派生（0.1 节的定性），"真实传感器有噪声、会掉线"是导师和
审稿人必然要问的问题。我们预注册的进入 P2 的门槛是：噪声 std=0.05 时保住 ≥60% 的干净增益，
否则整条路线停止。做法：对冠军配方的绳输入注入高斯噪声（std 以归一化绳长为单位，括号内为近似
毫米换算）与随机掉线（dropout = 每个读数以该概率失效），在 mask70 上重跑：

| 噪声 std（约合 mm） | 掉线率 | 全关节增益（mm） | 保留率 | 干净指尖增益（mm） |
|---:|---:|---:|---:|---:|
| 0 | 0 | -1.68 | 100% | -0.24 |
| 0.025（~1.2mm） | 0 | -1.60 | 95% | -0.22 |
| 0.05（~2.5mm） | 0 | -1.38 | 82% | -0.16 |
| 0.05 | 0.2 | -1.01 | 60% | – |
| 0.075（~3.8mm） | 0 | -1.09 | 65% | -0.10 |
| 0.10（~5mm） | 0 | -0.78 | 46% | – |
| 0.15（~7.5mm） | 0 | -0.19 | 11% | **+0.10（有害）** |
| 0.20（~10mm） | 0 | +0.38（有害） | 失效 | – |

（保留率 = 该行全关节增益 / 无噪声增益 -1.68 mm。）

读法与结论口径：**中等噪声（≤2.5 mm）保住 >80% 增益**——预注册门槛（≥60%）通过，2.5mm 噪声 +
20% 掉线的组合恰好压在 60% 线上；**有用性边界在 ~5–7.5 mm**；再往上修正转为有害，且伤害最先
出现在干净指尖上（std=0.15 行干净指尖已 +0.10）——噪声大到一定程度，传感器开始"冤枉"本来
预测正确的手指，这与 2.3 的有害闭合机制自洽。

这张表同时是**硬件规格书**：真实绳传感器（第 7 节阶段 1）的测距精度需达到 ~2.5 mm 级，
5 mm 是勉强可用的下限。本表只覆盖**零均值抖动与掉线**；真实设备的另一类主要误差——标定
偏移与尺度失配——见下面的 3.1（E1 实验，2026-07-09 完成）。

> **【插图位 2：noise_curve.png】**
> 文件：`docs/assets/rope_report/noise_curve.png`
> 图注建议：横轴为绳传感器噪声 std（归一化单位），纵轴为全关节 PA 改善量（mm），两条线分别为
> 掉线率 0 / 0.2。读法：增益随噪声平滑衰减、无悬崖，约 0.15–0.2 处穿越零线转为有害；曲线的
> "缓坡"形状本身是部署友好的性质（传感器精度换增益是连续的权衡，不是全有全无）。

### 3.1 标定误差：零点偏移与尺度失配（E1，2026-07-09）

上表的噪声是**逐读数独立的抖动**；真实设备还有另一类误差——**标定没做准**：每根绳的零点整体
偏掉（偏移）、手大手小导致的比例失配（尺度）。这类误差是系统性的，而且我们事先最担心它——
门控靠残差判断"该不该修"，系统性偏移会制造假残差。E1 对教师与**发布学生**注入三种形态的
标定级扰动（mask70，扰动单位同噪声表，0.05 ≈ 2.5 mm）：

| 扰动形态 | 教师增益（保留率） | 发布学生增益（保留率） |
|---|---:|---:|
| 随机偏移 std 0.025 | -1.60（95%） | -1.61（98%） |
| 随机偏移 std 0.05 | -1.38（82%） | **-1.54（94%）** |
| 随机偏移 std 0.075 | -1.09（65%） | -1.41（86%） |
| 随机偏移 std 0.10 | -0.78（46%） | -1.22（75%） |
| 同号零点偏移 +0.05（全绳读长） | -1.68（≈100%） | -1.61（98%） |
| **同号零点偏移 −0.05（全绳读短）** | -1.27（75%） | -1.50（92%） |
| 尺度失配 ±5% | -1.62（96%） | -1.62（99%） |
| 尺度失配 ±10% | -1.45（86%） | -1.56（95%） |
| 偏移 0.05 + 尺度 5% 组合 | -1.33（79%） | -1.51（92%） |

（保留率：教师对其干净增益 -1.68，学生对发布增益 -1.636。口径说明：**逐样本随机偏移在单帧
协议里与高斯噪声数学等价**——教师行数字与上面噪声表逐位一致，属交叉验证而非新信息；本表真正
新增的是**同号偏移、尺度失配、以及学生路径的全幅度扫描**。）

四个结论：

1. **预注册门槛大幅通过**：学生在偏移 0.05 下保留 94%（门槛 60%）——**不需要增广重训**，
   真机验证可以直接用发布学生。
2. **学生比教师更耐标定误差，且这是"被反驳的预测"**：赛前预测是"学生没见过系统性误差，退化
   应不慢于教师"；实测学生全线更稳（偏移 0.10 处 75% vs 46%）——噪声增广学出的平滑映射泛化
   到了它没见过的扰动形态（偏移、尺度）。蒸馏的去噪红利第三次出现（前两次：噪声下反超教师、
   多教师去噪）。
3. **系统性扰动确实会把门骗开，但学生仍稳**（另一条被反驳的预测）：赛前担心假残差导致误修；
   实测 gated fraction 随扰动单调上升（0.406 → 0.490），说明更多手指被允许修正，教师也确实
   明显退化；但学生退化更慢，靠的是蒸馏后的平滑/保守 α 映射，而不是门控"多关"。
4. **同号偏移有符号不对称性，直接写进硬件标定守则**：全绳读长（+0.05）几乎无损，**全绳读短
   （−0.05）是最坏情形**——基线错误本来就是"手指太张开"，读数再往"更弯"偏就会驱动过度弯曲、
   冤枉正确的手指。守则：**零点标定宁可偏向读长，不要读短**（如在张平手位置标零）。

## 4. 第四幕：蒸馏学生——把 400 步优化压成一次前向

### 4.1 为什么蒸馏、怎么蒸（SPIN 类比）

教师（第二幕冠军）每个样本要做 400 步优化，每步一次 MANO 前向+反向——效果好但根本无法实时。
自然的下一步是把优化的输出**蒸馏**成一个前向网络。类比 SPIN（ICCV'19）：SPIN 把 SMPLify 迭代
拟合的结果作为监督喂给回归网络，让网络学会优化器的解；我们同构地把测试时绳优化得到的 α 作为
标签，训练一个直接从输入预测 α 的学生网络。蒸馏还有一个已知副作用（SPIN 一脉的经验同样成立）：
学生学到的是大量逐样本解的**平滑平均**，比逐样本优化更稳——这个副作用在 4.2 的噪声行里会
变成可测量的优势。

学生架构刻意保持最小：**输入 65 维**（基线姿态 + 绳读数 + 绳残差 + 有效位）→ MLP
65→256→256→15 → **输出 15 个 α**，之后走与教师完全相同的 flex15 动作应用、gate010 硬门控与
MANO 解码。遵守可观测性原则：学生只输出绳能决定的量（15 个限幅标量），门控保持硬规则不学习。

### 4.2 严格 train→eval 分离下的恢复率

**协议**：学生只在 FreiHAND mask70 **训练** split 的教师标签上训练（32,560 样本）；下表每个
数字都来自 held-out **评测** split，且后四行分别是学生**从未见过**的扰动类型 / 数据集 / backend /
噪声条件。恢复率 = 学生增益 / 教师增益（预注册门槛：≥80% 才算蒸馏成功）。

| 评测格 | 学生增益（mm） | 教师增益（mm） | 恢复率 |
|---|---:|---:|---:|
| FreiHAND mask70（同分布） | -1.64 | -1.68 | **97%** |
| FreiHAND finger_end80（跨扰动） | -1.60 | -1.66 | 97% |
| HO3D v2 mask70（跨数据集） | -0.91 | -1.05 | 87% |
| HaMeR mask70（跨 backend） | -1.65 | -1.70 | 97% |
| mask70 + 传感器噪声 0.05 | **-1.53** | -1.38 | **110%（反超教师）** |

读法：前四行说明一次前向的学生几乎不损失教师的能力，且泛化模式与教师一致（跨数据集同样是最弱
轴，87%）。**最后一行是本幕最有信息量的结果**：在带噪传感器下学生反超教师。机制：学生训练时对
绳输入做噪声增广 + 蒸馏本身的平均化效应，使它学到对噪声鲁棒的平滑映射；而教师是逐样本优化，
会忠实地把噪声"优化进去"。这说明学生不只是教师的廉价近似——在真实（带噪）部署条件下它是
**更好**的选择。遮挡指尖切片同样保持：学生 -5.26 mm vs 教师 -5.47 mm。

### 4.3 对照与稳定性（学习式变体的强制检查清单）

任何学习式变体我们强制跑四件套：shuffle 对照、验证集早停、传感器噪声增广、多 seed。本轮结果：

| 检查 | 结果 | 说明 |
|---|---|---|
| **shuffle 对照**（样本间打乱绳读数重训学生） | 增益塌缩到 -0.05 mm，闭合率 0.7%，α 幅值 0.0016 | 最关键的反捷径对照：如果学生是在背"遮挡样本的姿态先验"而不是读绳，打乱绳读数不应伤增益。实际增益归零 → **增益可归因于绳信号本身** |
| seed 0/1/2 | PA 8.432 / 8.436 / 8.438 mm（±0.003 mm） | 结果对随机性不敏感，学生超参未经调优（呼应第 6 节弱点 2） |
| clean split | 学生 4.979 vs 基线 4.956 mm = +0.023 mm，在 ±0.05 mm 中性带内 | 学生在干净图上无害（教师带门控时是 +0.10 mm，学生反而更收敛）；但定性是"hard 样本修正器，clean 中性"，不宣称 clean 改进（弱点 4） |
| 无增广变体 | 干净绳输入下 -1.72 mm（略超教师），但噪声 0.05 下差于增广版 | 见 4.4 的部署默认决策 |

### 4.4 发布模型：四教师多样化学生

**转向理由**：单教师学生的短板清晰——跨数据集轴只有 87%。既然学生吃的是教师标签，直接的假设
是：**教师标签的多样性决定学生的泛化**。于是用四个训练教师联合训练一个学生：FreiHAND mask70 /
FreiHAND finger_end80 / HaMeR mask70 / HO3D v3 mask70（共 118,512 教师样本），架构与增广策略
完全不变。加入 HO3D 教师前先做了质量体检：HO3D v3 训练教师的闭合率 0.582 / 门控率 0.282 / α
幅值 0.043，与 FreiHAND 训练教师（0.504 / 0.288 / 0.053）处于同一工作区间——这个"教师质量对齐检查"
在 4.5 会被证明是必要的纪律。

| 评测格 | 基线 PA（mm） | 多教师 PA（mm） | 多教师增益 | 单教师 PA（mm） | 单教师增益 | 裁决 |
|---|---:|---:|---:|---:|---:|---|
| FreiHAND mask70 | 10.07 | 8.43 | -1.636 | 8.43 | -1.636 | 持平——主结果无损 |
| FreiHAND finger_end80 | 9.98 | 8.35 | -1.623 | 8.37 | -1.603 | 略好 |
| HO3D v2 mask70 | 9.44 | **8.46** | **-0.972** | 8.52 | -0.911 | +0.06 mm（恢复率 87% → 93%） |
| HaMeR mask70 | 10.82 | **9.13** | **-1.697** | 9.17 | -1.655 | 达到 HaMeR 教师的 **100%** |
| mask70 + 噪声 0.05 | 10.07 | 8.53 | -1.539 | 8.54 | -1.527 | 略好 |
| shuffle 对照 | 10.07 | 10.01 | -0.056 | – | – | 仍然塌缩 |
| clean split | 4.96 | 5.00 | +0.041 | 4.98 | +0.023 | 仍在 ±0.05 中性带内，但更靠近带边缘——如实说明 |

（"基线 PA"为该评测格不做任何修正的同解码器 PA；各格基线不同——遮挡类型、数据集、backend
不同，损伤本来就不同——所以跨行比较看"增益"列，行内看绝对值。）

**部署默认的决策**（增广 vs 无增广）：无增广学生在无噪声评测上略好，但在噪声 0.05 格上只保留
其干净增益的 82%（-1.414 mm），增广学生保留 93%（-1.527/-1.539 mm）。**增广用 0.08 mm 的干净
精度换噪声鲁棒性**；由于部署场景必然是带噪真实传感器，我们选增广版为发布模型。
**发布模型 = 四教师、增广、65 维输入 → 15 α（flex15 + gate010）的多样化学生。**

### 4.5 负结果消融：第五个教师反而有害（multi-v2）

**动机与预期**：四教师有效之后，顺理成章的假设是"第五个教师继续帮"。我们加入 HO3D v3
finger_end80 训练教师（总教师样本 139,344）训练 multi-v2 学生：

| 评测格 | multi-v2（mm） | 四教师发布版（mm） | 裁决（对照 seed 噪声 ±0.003） |
|---|---:|---:|---|
| FreiHAND mask70 | -1.636 | -1.636 | 统计打平 |
| FreiHAND finger_end80 | -1.619 | -1.623 | 统计打平 |
| mask70 + 噪声 0.05 | -1.536 | -1.539 | 统计打平 |
| HaMeR mask70 | -1.685 | -1.697 | 边缘性略差（差 0.012） |
| HO3D v2 mask70 | -0.834 | -0.972 | **唯一真实回退（0.14 mm，远超 seed 噪声）** |
| HO3D v2 finger_end80（新轴） | -0.524 | – | 正但偏小 |
| shuffle 对照 | -0.059 | -0.056 | 仍塌缩 |

先把现象摆准：multi-v2 **不是**"整体变差"——多数格在 seed 噪声内打平。需要解释的是一个精确
的现象：**为什么第五个教师只伤了 HO3D mask70，而 FreiHAND 各格毫发无损？**

**先排除一个直觉解释**（"两个 HO3D 教师共享底图所以互相干扰"）：反例就在四教师版里——
FreiHAND mask70 / finger_end80 两个教师同样共享同一批 32,560 张底图，却相安无事。"同图不同
扰动"不是病因。真正的差别在**教师质量**：

| 训练教师（学生的标签来源） | 闭合率（收敛质量代理） |
|---|---:|
| FreiHAND mask70 | 0.504 |
| FreiHAND finger_end80 | 0.525 |
| HaMeR mask70 | 0.508 |
| HO3D v3 mask70 | 0.582 |
| **HO3D v3 finger_end80（第五个）** | **0.565** |

远程 run 精确回填：`/data/wentao/ropetrack/runs/rope_p2_multi_v2_20260707_190953/r1_ho3d_v3_finger_train/teacher/rope_flex15_gate010/summary.json`。
这排除了一个错误解释：第五教师**不是欠收敛坏教师**。它的闭合率与前四个处在同一健康区间；
multi-v2 的问题更窄，是把一个 HO3D finger_end 专门教师混进通用学生后，挤压了 HO3D mask70
所在输入区域的条件均值。

**机制，三步**：① 学生看不见图像、也不知道样本来自哪个数据集——它只是 65 维输入（姿态+绳+
残差）到 α 的单值函数。教师之间的"冲突"不发生在图像层面，发生在 **65 维输入空间的重叠区域**：
同一片输入，两个教师给出不同的目标 α，学生只能学两者的**条件均值**。② 第五教师在自己的
finger_end 轴上是正收益但偏小（HO3D v2 finger_end80 只到 -0.524 mm），说明它教的是一个更专门、
更保守的局部修正策略；这个策略平均进 HO3D 的重叠区域后，会把 mask70 的条件均值从原来的通用
修正方向拉开，HO3D mask70 的增益因此缩水。③ FreiHAND 为什么
不受伤：它的姿态统计（演示性张手姿态）与 HO3D（抓握姿态）在输入空间里是可分的两片区域，毒素
是局部的漂不过去；且 FreiHAND 系教师占 3/5 数据，轻微重叠处也是它说了算。顺带解释新轴为何仍
是正的：-0.524 约为该 split 损伤（约 1.4 mm）的 1/3 强，说明新增教师确实教到东西，只是不值得
用 HO3D mask70 的回退来换。

**决定性证据**：把**发布模型**（从未见过 HO3D finger_end80 教师）零改动放到同一条新轴上评测，
得到全关节 -0.586 mm / 遮挡指尖 -0.952 mm——**反超**专门为这条轴加了教师的 multi-v2（-0.524 /
-0.735 mm）。加这个教师，在它自己的主场都没赢。

**裁决与修法**：保留四教师版为发布模型，multi-v2 记为负结果消融。原则：**教师多样性有益的前提
不仅是教师能收敛，还要不在同一输入区域给通用学生互相拉扯的标签**。修法三条：① 踢掉这个
专门教师（已做）；② 给学生一个能区分"这是哪种损伤"的信号——65 维输入分不清 mask 伤与指端伤，
而图像一看便知：这才是图像特征（P3 方向）合理的角色定位，见第 7 节；③ 扩教师/扩数据集前先做
held-out 轴的负迁移体检，而不是只看教师 closure。

### 4.6 运行时开销（口径必须分开说）

| 路径 | 每样本计算 | 全评测集墙钟（3,960 样本，batch=512） | 摊薄每样本 | 逐帧口径（batch=1） |
|---|---|---:|---:|---|
| 教师（400 步优化） | 400 次 MANO 前向+反向 | 110.32 s | 27.9 ms | **秒级**/帧 |
| 发布学生 | 一次 65→256→256→15 MLP 前向 | 51.66 s | 13.0 ms | **毫秒级**/帧 |

两个口径都成立，但汇报时不可混用：① 27.9 / 13.0 ms 是 **batch=512 摊薄的端到端 apply 管线
墙钟**，含缓存加载、动作应用、MANO 解码与写盘（MANO 解码两条路径共享、各做一次），因此它
**低估**纯 α 网络的加速比，是保守数字，**不是逐帧延迟**；② 逐帧 demo（batch=1）口径下，教师
每帧要串行做 400 步优化，为秒级/帧，学生一次小 MLP 前向为毫秒级/帧——实时应用只有学生路径
可行。

## 5. 第五幕：天花板与剩余余量（oracle 与 E2 剪刀图）

**oracle 是什么、为什么要跑**：把优化目标从"绳读数"换成 **GT 指尖位置**，其余（动作空间、
优化器、强配方）完全不变。这给出"该动作空间 + 该优化器"能达到的**上限**——余量 = oracle 增益
− rope 增益，即换更好的信号或方法后，在同一动作空间里**最多**还能挤出多少。它同时是防泄漏
证据：如果 5 个绳标量已经"泄漏"了答案，rope 就应贴着 oracle；实测存在稳定差距，且 5 标量
相对 63 维姿态答案（21 关节 × 3）信息量极小——不存在平凡泄漏。（E2 又补充了两样：
oracle_chain = 以全部 20 个非腕关节为目标；pose45 = 45 维逐关节全 pose 增量动作空间，见
5.1。）下表先给各评测轴在 **flex15 空间内**的天花板：

| 评测设置 | 基线 PA（mm） | oracle 后 PA | oracle 增益 | rope 后 PA | rope 增益 | 净余量（mm） | rope 达到 |
|---|---:|---:|---:|---:|---:|---:|---:|
| FreiHAND mask70 | 10.07 | 8.10 | -1.97 | 8.39 | -1.68 | 0.29 | 85% |
| FreiHAND finger_end80 | 9.98 | 7.50 | -2.48 | 8.32 | -1.66 | 0.82 | 67% |
| HO3D v2 mask70（WiLoR） | 9.44 | 8.28 | -1.16 | 8.39 | -1.05 | **0.11** | **90%** |
| HO3D v2 mask70（HaMeR） | 9.32 | 8.03 | -1.29 | 8.67 | -0.66 | 0.63 | 51% |

（绝对值列由源表 cm 值换算，含 ±0.01 mm 舍入。）

**措辞纪律（重要）**：在 HO3D/WiLoR 上，绳修复已拿到 **flex15 空间内** oracle 的 90%，净余量
只有 0.11 mm——这条轴在该空间内已接近天花板，不能写成"后续改进空间很大"。任何图像条件化修复
（P3 方向）的论据只能建立在确有余量的轴上：finger_end80（0.82 mm）与 HaMeR（0.63 mm）。
（pose 空间层面的天花板只在 mask70 上测过，见 5.1，不外推到 HO3D。）

### 5.1 剪刀图（E2，2026-07-09）：动作空间加大，oracle 涨、rope 不涨

上表的"天花板"是 flex15 空间内的。自然的追问是：**换更大的动作空间，天花板还能抬多高？
绳跟得上吗？** E2 在三个维度（5/15/45）上各跑两遍同一强配方——绳驱动 vs 真值驱动：

| 动作空间（维度） | rope 增益（mm） | oracle_tip 增益 | oracle_chain 增益 |
|---|---:|---:|---:|
| mult5（5） | -1.55 | -1.83 | -1.59 |
| flex15（15，发布配置） | -1.68 | -1.97 | -1.83 |
| pose45（45，逐关节全 pose 增量） | -1.77 | -3.15 | **-3.39** |

（FreiHAND mask70；rope 行带 gate010、oracle 行不带，与上表口径一致。正则敏感性检查：pose45
的 alpha_l2 放大 4 倍结果逐位不变——45 维 oracle 不是正则化假象。）

> **【插图位 5：剪刀图】**
> 文件：`docs/assets/rope_report/e2_action_space_scissors.png`。

读法：**剪刀如预注册预期张开**。rope 线走平——pose45 只比 flex15 多 0.09 mm，落在预注册的
±0.15 mm 判定带内：5 根绳驱动不了更多自由度，发布配置维持 flex15 不变（pose45 的跨轴安全性
也未测）。oracle 线大开——45 维下 oracle_chain 达 -3.39 mm，遮挡指尖改善约 -11.6 mm：
**遮挡失败在 pose 空间里大部分是可表达的，卡脖子的是信息，不是修正模块**。一个方法学细节：
15 维时 tip 目标优于全关节目标（-1.97 vs -1.83，小空间盯住损伤集中地更有效），45 维时反转
（-3.39 vs -3.15，空间大了 5 个指尖钉不住 45 个自由度）——"传感信息解锁动作空间"的又一佐证。

由此把 mask70 的 5.07 mm 损伤完整分解成三段（"为什么 oracle 增益远小于损伤"这个组会必问
的问题，答案就是这张分解）：

| 分量 | 大小 | 含义 |
|---|---:|---|
| 5 根绳已修掉 | 1.77 mm | rope/pose45 实测（发布配置 flex15 为 1.68） |
| **信息缺口** | **1.62 mm** | 在 pose 空间内、但 5 根绳的信息驱动不了（oracle_chain/pose45 − rope/pose45）——"更丰富传感/信息"的定价，即第 7 节阶段 3 各分支追逐的预算 |
| pose 空间之外 | 1.68 mm | 连 45 维有界 pose 修正也表达不了（手形 β、GT 噪声、不可约部分） |

要抬天花板只有两条路：给系统**更多信息**（更多绳 / 时序 / 局部图像证据），或让网络端到端吃
绳信息（rope-LoRA）。"把修正模块做大"这条路已被 E2 正式排除。

## 6. 必须主动交代的弱点（前 5 条为 P2 收官登记，后 2 条为本次复盘新增）

以下各条在任何对外表述中主动交代，不等被问：

1. **模拟传感器**：绳读数由 GT 关节派生；第 3 节的噪声/标定消融就是现实性边界。5 个标量携带
   的信息远少于 63 维姿态答案（第 5 节的 oracle 差距证明不存在平凡泄漏）。硬件原型已制成，
   真实读数验证是第 7 节阶段 1–2 的核心。
2. **配方来源**：优化器超参（steps/lr/正则）是在评测 split 上选出的。缓解措施：该配方在 HO3D、
   finger_end80、HaMeR 上零重调迁移成立；学生侧超参未经调优，且 seed 稳定性 ±0.003 mm。
3. **协议边界**：所有增益均为同解码器口径；不得与早期硬基线表（0020，各 backend 原生解码协议）
   在关节级拼接比较（存在 ~0.3–0.5 mm 协议偏移）；跨表比较只用 mesh/F 类指标。
4. **干净图像**：本管线定性为 hard/遮挡样本修正器；clean split 上学生是中性（+0.02 mm），
   **不是**改进。
5. **有效教师多样性**：四个训练教师中的三个 FreiHAND 系教师共享同一批 32,560 张底图（区别只在
   扰动类型/backend）；数据集层面的多样性是 **2 个语料库**，不是 4 个。
6. **标定误差已消融、真实误差谱未实测（E1 后更新）**：E1 已覆盖零点偏移/尺度失配（3.1），
   结论良性——发布学生在标定级扰动下保留 92–99%、免重训；扰动会打开更多 gate，但学生仍比
   教师稳。残余弱点是**真实设备的误差谱**（滞后、接触噪声、佩戴差异）尚未实测对齐——这是
   第 7 节阶段 2 的表征任务。
7. **训练姿态分布窄（新增）**：学生只见过 FreiHAND（演示性姿态）与 HO3D（抓握）的教师标签，
   连续动态手势下的分布外行为未验证。结构性兜底：输出有界（tanh ≤0.5）+ 硬门控 + clean 中性
   ——最坏情形是"少修"，不是"修坏"。根本解法是免标注再蒸馏：教师优化不需要 GT，可在真实
   采集数据上自生成标签做个人化适配（第 7 节阶段 3）。

## 7. 下一步：分阶段真机验证路线（每阶段带判据，请导师把关）

P2 闭环、E1/E2 收官之后，全篇尚未验证的主张只剩一条：**真实绳读数能否复现仿真里的有效信号**。
所以下一步不是并列多选题，而是一条分阶段、带判据、按结果分流的路线。排序的理由（也是为什么
时序不放第一步）：当前的失败模式是"**单帧信息不够**"，不是"时序不一致"——E2 已证明 5 绳配
更大动作空间也确定不了更多自由度；时序平滑能去抖动、补一点动态先验，但不会凭空变出 PIP/DIP/
侧摆信息。真机单帧链路成立之前，时序没有验证对象。

### 阶段 1：受控真机 mini-demo（绳线手套 + 腕部 RGB 合体）

- **目的**：不追指标，只验证"真实 rope 读数能否复现 sim 的有效信号"——导师最关心的现实性
  问题一步到位。
- **做法**：少量固定手势、多段录制，同步腕部 RGB 与绳读数；简单标定（张平手/握拳两姿势），
  并遵守 E1 守则——**零点宁可让绳读长、不要读短**（全绳读短是实测最坏的系统性情形，3.1）；
  离线跑发布学生与教师 apply，看可视化与绳残差闭合是否按 sim 预期发生。
- **判据**：固定手势下真实绳残差方向与手指弯曲方向一致；简单标定后的偏移/尺度/噪声落进 E1
  坐标系；离线 apply 后残差闭合与 sim 同量级，且遮挡时段可视化不劣化。E1 已保证：只要真实误差
  落在偏移 0.05 / 尺度 10% 量级内，发布学生**免重训**即可用（保留 92–95%）。

### 阶段 2（demo 信号成立后）：真实误差谱表征 + 标定方式选型

- 小规模采集真实 rope/RGB 配对数据，**先表征、不训练**：估计真实偏移、尺度、滞后、接触噪声
  的分布，逐项对齐到 E1/噪声曲线的坐标系上，得出"真实设备处在容忍带的哪个位置"。
- 同时回答"**怎么标定才能配合 refine 模块效果最好**"，三个候选对比：① 两姿势参考标定；
  ② 视觉锚定在线标定——手清晰可见时基线预测很准（clean 4.99 mm），用它反推预测绳长、对原始
  读数逐指回归零点/尺度并持续更新，遮挡来临时用最新标定接管（视觉与绳失效模式互补，本身可作
  方法贡献）；③ 时序差分读数（差分天然免疫零点偏移）。
- **判据**：至少一种标定方式把真实误差压进 E1 的 ≥80% 保留带。

### 阶段 3（按阶段 2 的结果分流，三选一）

- **真实误差谱 ≈ E1 范围** → 学生不重训，进入**扩数据**：加新数据集（首选 DexYCB）检验修正
  是否只是"小范围动作的拟合"——当前对泛化性的主要疑虑（弱点 7）；配套免标注再蒸馏（教师自
  生成标签）做个人化适配；教师质量体检（4.5 纪律）为硬前提。
- **误差谱更复杂**（重尾、滞后主导）→ 启动 E1 Phase-2：bias/scale 增广重训学生（规格已备，
  门槛写死在 spec 里）。
- **误差主要是单绳信息不足**（表现为卡在 E2 定价的 1.62 mm 信息缺口上）→ 进入 E3：每指第二
  根绳的仿真定价（R10-PIP 布线 / 相邻指尖距探侧摆），先 A/B 附着点再动硬件。动机：单绳每指
  只给 1 个约束而弯曲有 3 个自由度，PIP/DIP 在勾握、按压类抓握中解耦；E2 的 45 维 oracle
  （-3.39）就是这条分支的收益上界。

### 阶段 4：时序（有连续真实数据之后才有意义）

- 先做懒版本：对绳读数或 α 做时域滤波 / 卡尔曼级平滑——若能显著降抖再考虑学习式时序模型，
  不一上来搭时序网络。
- 学习式版本的两个组件已想好：从高置信度稳定时段锁定手形 β 与参考 MANO、逐帧只修 pose；
  时序差分读数消零点漂移（与阶段 2 的在线标定同一机制）。
- 届时评测协议迁移：FreiHAND（无时间轴）退役为训练/消融，主评测改 HO3D/DexYCB 的"遮挡
  episode"（遮挡时段误差 + 结束后恢复速度），更贴遥操真实场景。

### 降级说明：P3 v1（token 图像特征）不作为主线

P3 v0 用 pooled 全局特征无增益且 image-only 对照归零（rope-only -1.636 / 加 pooled 特征
-1.533 / 纯图像 -0.100 mm）。E2 进一步确认瓶颈在**信息**：图像 token 理论上也是信息源，但
v0 只证伪了 pooled 全局特征，不足以证明局部 token 一定无用；候选信息源的证据排序暂定为
**更多绳（直接）> 时序（免费）> 图像 token（待证）**。4.5 的机制分析指出图像特征更合理的角色
是"区分损伤类型、决定该信绳多少"
的门控信号。**降为可选消融**。更大的训练式变体（遮挡增广微调 baseline 与 rope-LoRA 端到端
条件化）共享一套训练基建，待阶段 2 的真实数据落地后一起立项——微调 baseline 同时是审稿人
必问的对照。

**一句话版**：先证明真实链路成立（阶段 1–2），再按证据分流"扩数据 / 重训 / 换传感器"
（阶段 3），时序垫后（阶段 4）。每一步的判据事先写死，E1/E2 已为所有分支定好价。
