## 0. 任务设定与评价口径（术语先行，后文直接引用）

### 0.1 任务与"绳长"信号

单目 RGB 手姿态估计（输出 MANO 参数化手模型的 21 个关节 3D 位置）在**手部被遮挡时显著退化**。
我们的设想：在手套/腕带形态的设备上布 5 根从各指尖到手腕锚点的"绳"，实时读出 5 个长度标量；
用这 5 个标量在**测试时**修正视觉模型的预测，不改动、不重训视觉模型本身。

**必须先行声明的定性（贯穿全文）**：当前阶段的绳长读数是**由 GT 关节位置派生的模拟传感器**，
不是真实硬件。它的现实性边界由第 3 节的噪声/掉线消融给出；"5 个标量远少于 63 维姿态答案、
不构成平凡泄漏"由第 5 节的 oracle 差距证明。凡对外表述，此定性必须出现。

### 0.2 指标与术语

**PA-MPJPE（mm，全文主指标）**：预测与 GT 的 21 关节平均位置误差，计算前先做 Procrustes
对齐（求最优旋转+平移+缩放后再比较）。它度量"姿态形状"错误——手指到底弯成什么样——而不含全局
位置/朝向/尺度误差。选它的理由：遮挡损伤恰恰集中在"手指弯曲状态估错"这类形状错误上，PA 口径
把问题聚焦在这里。数值越低越好。

其余术语集中列出，后文不再重复解释：

| 术语 | 含义 |
|---|---|
| 增益 / delta（mm） | 修复后 PA 减修复前 PA，**负 = 改善**。基线与修复后走**同一个** MANO 解码管线（"同解码器"口径），避免协议差异污染差值 |
| occluded-tip / clean-tip 切片 | 只在"被遮挡指尖 / 未遮挡指尖"子集上统计的增益。遮挡指尖是损伤集中地，也是绳信号应当起作用的地方 |
| 闭合率（closure） | 优化（或学生）把"预测绳长 vs 传感器读数"之间残差消除掉的比例。注意：闭合率只衡量"服从传感器"的程度，**不等于姿态变好**（见 2.3 的"有害闭合"证据） |
| 动作空间 mult5 / flex15 | 修正量 α 的参数化：mult5 = 每指 1 个弯曲乘子（5 标量）；flex15 = 每指 3 个关节各一个弯曲增量（15 标量）。α 经 tanh 限幅，学习式变体最后一层零初始化 |
| 门控 gate010 | 硬规则（非学习）：绳残差 < 0.1（归一化单位）的手指不做任何修正（α 置 0）。gated fraction = 被门控关闭的手指占比 |
| hard split | 程序化生成的遮挡评测集：mask70 = 手部区域约 70% 被遮挡块覆盖；finger_end80 = 每根手指远端 80% 被遮挡（五指全部受累）。程序化生成保证与 clean 版本严格同图成对，可做受控对照 |
| backend | 被修复的视觉模型：WiLoR（主 backend）与 HaMeR（第二 backend，用于验证结论不是 WiLoR 特有）。全部在 GT 检测框下评测，排除手部检测器差异 |
| 数据集 | FreiHAND（训练劣化 split 32,560 样本 / 评测劣化 split 3,960 样本）；HO3D v2（跨数据集评测）与 HO3D v3（训练数据来源），HO3D 为手—物交互视频 |

**决策纪律**（贯穿各阶段）：每个实验在开跑前注册可证伪的预测与数值门槛，跑完对照门槛裁决；
负结果记录在案而不丢弃。文中每个"转向"处会给出当时的判断依据。

## 1. 第一幕：遮挡打垮 RGB baseline（+5.07 mm）

| 量 | 值 |
|---|---:|
| FreiHAND clean PA-MPJPE（WiLoR，GT bbox） | 4.99 mm |
| FreiHAND mask70 hard PA（同解码器基线） | 10.07 mm |
| **遮挡损伤** | **+5.07 mm** |

读法：同一个模型、同一批底图，只是加上程序化遮挡，误差就**翻倍**（4.99 → 10.07 mm）。这确立了
问题的规模，也定义了后文一切"修复"的分母——我们汇报修复效果时统一说"修复了 5.07 mm 损伤中的
多少"。

**失败模式诊断**（绳信号选型的依据）：对逐样本误差做诊断分析发现，损伤集中表现为**被遮挡的蜷曲
手指被预测得过于张开**——视觉证据缺失时，模型回退到"手指伸直"的先验。这直接指出了对症的修正
方向：手指的**弯曲程度**。而"指尖—手腕距离"（绳长）恰好是弯曲程度的单调观测量——手指越蜷，
指尖离手腕越近。这就是为什么第二幕的修正动作空间选"弯曲标量"（mult5/flex15），而不是任意的
姿态修正。

**设计说明（为什么用程序化 hard split 而不是真实遮挡数据）**：真实遮挡数据没有"同图去遮挡"的
成对参照，无法把"遮挡损伤"从数据分布差异中剥离出来；程序化遮挡给出严格成对的 clean/hard 对照，
且严重度可控（mask70 与 finger_end80 是两类结构不同的遮挡：面积型 vs 指端型）。真实遮挡场景由
HO3D（手持物体的真实交互视频）作为跨数据集验证补上。

## 2. 第二幕：绳长测试时优化，修复约 1/3 损伤

方法：对每个测试样本，冻结视觉模型，只对 α（5 或 15 个弯曲标量）做梯度优化，目标是让修正后
姿态的**预测绳长逼近传感器读数**；α 应用后走同一 MANO 解码得到修正姿态。无任何训练、无任何
标注参与——这是纯测试时（test-time）修复。

**为什么把学习问题先退化成逐样本优化**：我们最早的学习式尝试（直接回归 45 维姿态修正的 MLP）
在训练缓存上拟合但在 held-out 评测上变差——教训是**输出空间超出了 5 个标量所能决定的范围**
（"可观测性原则"：学习模块只应输出绳能决定的量）。逐样本优化 + 低维弯曲动作空间是这一教训的
直接产物，后来它反过来成为第四幕蒸馏的教师。

### 2.1 剂量—响应诊断：小增益是"优化不足"，不是"信号没用"

**转向的理由**：P0 阶段默认优化配方下绳修复的增益很小（全关节仅 0.16 mm）。此时有两个竞争假设：
(a) 绳信号本身信息量不足；(b) 优化强度不够、残差根本没被消掉。区分它们的诊断证据是残差闭合率
——默认配方只闭合了 3.8% 的绳残差，即传感器的话**根本没被听进去**，指向 (b)。于是固定动作空间
（mult5，P0 默认）不变，只扫优化强度（步数 × 学习率），看剂量—响应关系：

| 配方 | 绳残差闭合率 | 全关节增益（mm） |
|---|---:|---:|
| lr=2, steps=120（P0 默认） | 3.8% | 0.16 |
| lr=8, steps=400 | 27.8% | 1.00 |
| lr=32, steps=120 | 30.1% | 1.07 |
| lr=32, steps=400 | 41.8% | 1.40 |

（本表"增益"列为改善幅度，正 = 改善；后文各表统一用带符号差值，负 = 改善。口径相同，仅符号
约定不同。）

读法：闭合率从 3.8% 提到 41.8%，增益同步单调地从 0.16 mm 涨到 1.40 mm。如果假设 (a) 成立，
加大剂量后增益应当很快平台化；实际是干净的单调剂量—响应关系——**信号是真的，此前只是没用上**。
这一步是整个项目的关键转折：它把"绳信号只值 0.16 mm"的错误结论翻转为后面所有结果的基础。

### 2.2 冠军配方与跨设置验证

在强优化配方下对动作空间 × 门控做栅格搜索，冠军为：

```
rope + flex15 + gate010 + steps=400 / lr=32 / alpha_l2=0.001
```

（复现口径说明：优化损失为 batch 均值，故有效学习率与 batch 大小耦合；所有公开结果固定
batch=512。）

**选择理由与一次被反驳的预测**：赛前我们注册的预测是"门控后的 flex15 不会赢 mult5"（理由：15
维空间由 5 个读数驱动，自由度过剩）。数据反驳了它——在两个 FreiHAND hard split 上 flex15+gate010
的 PA 都最优。反思后机制上说得通：flex15 的多余自由度被四重约束压住（α 的 L2 正则、tanh 限幅、
零初始化梯度下降的隐式最小范数偏置、残差门控）。我们按数据改判，且把这次反驳记录在案——预测被
推翻本身说明选型不是事后挑好看的数字。

冠军配方在五个评测设置上的成绩（**HO3D 与 HaMeR 均为零重调直接套用**，配方从未接触这些设置）：

| 评测设置 | PA（mm） | 全关节增益 | 遮挡指尖增益 | 闭合率 |
|---|---:|---:|---:|---:|
| FreiHAND mask70（WiLoR） | 8.39 | **-1.68** | **-5.47** | 48.5% |
| FreiHAND finger_end80（WiLoR） | 8.32 | -1.66 | -3.45 | 50.6% |
| FreiHAND mask70（HaMeR） | 9.12 | -1.70 | -5.64 | 51.3% |
| HO3D v2 mask70（WiLoR，零重调） | 8.39 | -1.05 | -2.09 | 54.2% |
| HO3D v2 mask70（HaMeR，零重调） | 8.67 | -0.66 | -0.82 | 55.4% |

读法（按行）：主设置 mask70 上全关节 -1.68 mm——占 5.07 mm 遮挡损伤的约 **33%**，且修复集中在
该集中的地方：遮挡指尖 -5.47 mm。第 2 行换遮挡类型、第 3 行换 backend，增益几乎不变（-1.66 /
-1.70），说明修复机制不挑扰动形态、不是 WiLoR 特有。第 4–5 行换数据集（真实手物交互视频）且
**完全不重调超参**，增益变小但方向一致——跨数据集是最弱的轴，这个观察贯穿后文（第四幕的多教师
改进、第七节的扩数据路线都由它驱动）。

**一句话总结第二幕**：一个 400 步的测试时优化 + 5 个标量的传感器读数，免训练地修复了约 1/3 的
遮挡损伤。

### 2.3 门控的机制证据："有害闭合"真实存在

为什么必须有门控、且门控是硬规则？对照实验给出机制证据：**不带门控**的 flex15 闭合了**更多**
绳残差（49.7% vs 48.5%），姿态却**更差**（-1.39 vs -1.68 mm）。也就是说，对绳残差本来就小的
手指（视觉预测已经和传感器一致）强行把残差压到零，会把正确的姿态拉坏——闭合率和姿态质量在这里
**解耦**。门控用"放弃一部分闭合"换取"不伤害无辜手指"。这也是我们把闭合率只当诊断量、从不当
优化目标汇报的原因。

## 3. 第三幕：传感器现实性——噪声与掉线

**为什么必须做这一节**：绳读数由 GT 派生（0.1 节的定性），"真实传感器有噪声、会掉线"是导师和
审稿人必然要问的问题。我们预注册的进入 P2 的门槛是：噪声 std=0.05 时保住 ≥60% 的干净增益，
否则整条路线停止。做法：对冠军配方的绳输入注入高斯噪声（std 以归一化绳长为单位，括号内为近似
毫米换算）与随机掉线（dropout = 每个读数以该概率失效），在 mask70 上重跑：

| 噪声 std（约合 mm） | 掉线率 | 全关节增益（mm） | 保留率 | 干净指尖增益（mm） |
|---:|---:|---:|---:|---:|
| 0 | 0 | -1.68 | 100% | -0.24 |
| 0.025（~1.2mm） | 0 | -1.60 | 95% | -0.22 |
| 0.05（~2.5mm） | 0 | -1.38 | 82% | -0.16 |
| 0.05 | 0.2 | -1.01 | 60% | – |
| 0.075（~3.8mm） | 0 | -1.09 | 65% | -0.10 |
| 0.10（~5mm） | 0 | -0.78 | 46% | – |
| 0.15（~7.5mm） | 0 | -0.19 | 11% | **+0.10（有害）** |
| 0.20（~10mm） | 0 | +0.38（有害） | 失效 | – |

（保留率 = 该行全关节增益 / 无噪声增益 -1.68 mm。）

读法与结论口径：**中等噪声（≤2.5 mm）保住 >80% 增益**——预注册门槛（≥60%）通过，2.5mm 噪声 +
20% 掉线的组合恰好压在 60% 线上；**有用性边界在 ~5–7.5 mm**；再往上修正转为有害，且伤害最先
出现在干净指尖上（std=0.15 行干净指尖已 +0.10）——噪声大到一定程度，传感器开始"冤枉"本来
预测正确的手指，这与 2.3 的有害闭合机制自洽。

这张表同时是**硬件规格书**：若做真实绳传感器（第 7 节路线 C），测距精度需达到 ~2.5 mm 级；
5 mm 是勉强可用的下限。

## 4. 第四幕：蒸馏学生——把 400 步优化压成一次前向

### 4.1 为什么蒸馏、怎么蒸（SPIN 类比）

教师（第二幕冠军）每个样本要做 400 步优化，每步一次 MANO 前向+反向——效果好但根本无法实时。
自然的下一步是把优化的输出**蒸馏**成一个前向网络。类比 SPIN（ICCV'19）：SPIN 把 SMPLify 迭代
拟合的结果作为监督喂给回归网络，让网络学会优化器的解；我们同构地把测试时绳优化得到的 α 作为
标签，训练一个直接从输入预测 α 的学生网络。蒸馏还有一个已知副作用（SPIN 一脉的经验同样成立）：
学生学到的是大量逐样本解的**平滑平均**，比逐样本优化更稳——这个副作用在 4.2 的噪声行里会
变成可测量的优势。

学生架构刻意保持最小：**输入 65 维**（基线姿态 + 绳读数 + 绳残差 + 有效位）→ MLP
65→256→256→15 → **输出 15 个 α**，之后走与教师完全相同的 flex15 动作应用、gate010 硬门控与
MANO 解码。遵守可观测性原则：学生只输出绳能决定的量（15 个限幅标量），门控保持硬规则不学习。

### 4.2 严格 train→eval 分离下的恢复率

**协议**：学生只在 FreiHAND mask70 **训练** split 的教师标签上训练（32,560 样本）；下表每个
数字都来自 held-out **评测** split，且后四行分别是学生**从未见过**的扰动类型 / 数据集 / backend /
噪声条件。恢复率 = 学生增益 / 教师增益（预注册门槛：≥80% 才算蒸馏成功）。

| 评测格 | 学生增益（mm） | 教师增益（mm） | 恢复率 |
|---|---:|---:|---:|
| FreiHAND mask70（同分布） | -1.64 | -1.68 | **97%** |
| FreiHAND finger_end80（跨扰动） | -1.60 | -1.66 | 97% |
| HO3D v2 mask70（跨数据集） | -0.91 | -1.05 | 87% |
| HaMeR mask70（跨 backend） | -1.65 | -1.70 | 97% |
| mask70 + 传感器噪声 0.05 | **-1.53** | -1.38 | **110%（反超教师）** |

读法：前四行说明一次前向的学生几乎不损失教师的能力，且泛化模式与教师一致（跨数据集同样是最弱
轴，87%）。**最后一行是本幕最有信息量的结果**：在带噪传感器下学生反超教师。机制：学生训练时对
绳输入做噪声增广 + 蒸馏本身的平均化效应，使它学到对噪声鲁棒的平滑映射；而教师是逐样本优化，
会忠实地把噪声"优化进去"。这说明学生不只是教师的廉价近似——在真实（带噪）部署条件下它是
**更好**的选择。遮挡指尖切片同样保持：学生 -5.26 mm vs 教师 -5.47 mm。

### 4.3 对照与稳定性（学习式变体的强制检查清单）

任何学习式变体我们强制跑四件套：shuffle 对照、验证集早停、传感器噪声增广、多 seed。本轮结果：

| 检查 | 结果 | 说明 |
|---|---|---|
| **shuffle 对照**（样本间打乱绳读数重训学生） | 增益塌缩到 -0.05 mm，闭合率 0.7%，α 幅值 0.0016 | 最关键的反捷径对照：如果学生是在背"遮挡样本的姿态先验"而不是读绳，打乱绳读数不应伤增益。实际增益归零 → **增益可归因于绳信号本身** |
| seed 0/1/2 | PA 8.432 / 8.436 / 8.438 mm（±0.003 mm） | 结果对随机性不敏感，学生超参未经调优（呼应第 6 节弱点 2） |
| clean split | 学生 4.979 vs 基线 4.956 mm = +0.023 mm，在 ±0.05 mm 中性带内 | 学生在干净图上无害（教师带门控时是 +0.10 mm，学生反而更收敛）；但定性是"hard 样本修正器，clean 中性"，不宣称 clean 改进（弱点 4） |
| 无增广变体 | 干净绳输入下 -1.72 mm（略超教师），但噪声 0.05 下差于增广版 | 见 4.4 的部署默认决策 |

### 4.4 发布模型：四教师多样化学生

**转向理由**：单教师学生的短板清晰——跨数据集轴只有 87%。既然学生吃的是教师标签，直接的假设
是：**教师标签的多样性决定学生的泛化**。于是用四个训练教师联合训练一个学生：FreiHAND mask70 /
FreiHAND finger_end80 / HaMeR mask70 / HO3D v3 mask70（共 118,512 教师样本），架构与增广策略
完全不变。加入 HO3D 教师前先做了质量体检：HO3D v3 训练教师的闭合率 0.582 / 门控率 0.282 / α
幅值 0.043，与 FreiHAND 训练教师（0.504 / 0.288 / 0.053）处于同一工作区间——这个"教师质量对齐检查"
在 4.5 会被证明是必要的纪律。

| 评测格 | 多教师学生（mm） | 单教师学生（mm） | 裁决 |
|---|---:|---:|---|
| FreiHAND mask70 | -1.636 | -1.636 | 持平——主结果无损 |
| FreiHAND finger_end80 | -1.623 | -1.603 | 略好 |
| HO3D v2 mask70 | **-0.972** | -0.911 | HO3D v3 训练数据带来 +0.06 mm（恢复率 87% → 93%） |
| HaMeR mask70 | **-1.697** | -1.655 | 达到 HaMeR 教师的 **100%** |
| mask70 + 噪声 0.05 | -1.539 | -1.527 | 略好 |
| shuffle 对照 | -0.056 | – | 仍然塌缩 |
| clean split | +0.041 mm | +0.023 mm | 仍在 ±0.05 中性带内，但更靠近带边缘——如实说明 |

**部署默认的决策**（增广 vs 无增广）：无增广学生在无噪声评测上略好，但在噪声 0.05 格上只保留
其干净增益的 82%（-1.414 mm），增广学生保留 93%（-1.527/-1.539 mm）。**增广用 0.08 mm 的干净
精度换噪声鲁棒性**；由于部署场景必然是带噪真实传感器，我们选增广版为发布模型。
**发布模型 = 四教师、增广、65 维输入 → 15 α（flex15 + gate010）的多样化学生。**

### 4.5 负结果消融：第五个教师反而有害（multi-v2）

**动机与预期**：四教师有效之后，顺理成章的假设是"第五个教师继续帮"。我们加入 HO3D v3
finger_end80 训练教师（总教师样本 139,344）训练 multi-v2 学生。结果否定了假设：

| 评测格 | multi-v2（mm） | 四教师发布版（mm） | 裁决 |
|---|---:|---:|---|
| FreiHAND mask70 | -1.636 | -1.636 | 持平 |
| HO3D v2 mask70 | -0.834 | -0.972 | **变差**——新教师没帮它想帮的轴，反而拖累近邻轴 |
| HO3D v2 finger_end80（新轴） | -0.524 | – | 正但幅度小 |
| HaMeR mask70 | -1.685 | -1.697 | 持平/略差 |
| shuffle 对照 | -0.059 | -0.056 | 仍塌缩 |

**决定性证据**：把**发布模型**（从未见过 HO3D finger_end80 教师）零改动放到同一条新轴上评测，
得到全关节 -0.586 mm / 遮挡指尖 -0.952 mm——**反超**专门为这条轴加教师的 multi-v2（-0.524 /
-0.735 mm）。即：加这个教师在它自己的主场都没赢。

**机制解读与裁决理由**：该新教师的收敛质量未达到其他四个教师的水平（这正是 4.4 那个"质量体检"
要拦的情况）。一个欠收敛、有偏的教师，在与现有教师输入分布重叠的区域里等价于**定向标签噪声**
——它不是补充信息，是污染。裁决：保留四教师版为发布模型，multi-v2 记为负结果消融。这个负结果
的价值在于确立了纪律：**教师多样性带来收益的前提是教师质量对齐**；后续任何扩教师/扩数据集的
动作（第 7 节路线 B）都先过教师质量体检。

### 4.6 运行时开销（口径必须分开说）

结构性事实：教师 = 每样本 400 次 MANO 前向+反向的优化；学生 = 一次 65→256→256→15 的 MLP 前向
（MANO 解码两者共享，各做一次）。

实测数字与其口径：在 FreiHAND mask70 全评测集（3,960 样本，batch=512）上，端到端 apply 脚本
墙钟时间为——教师优化路径 110.32 s（**摊薄** 27.9 ms/样本），发布学生路径 51.66 s（**摊薄**
13.0 ms/样本）。注意两点：① 这是 **batch=512 摊薄的管线墙钟**，含缓存加载、动作应用、MANO
解码与写盘，因此它**低估**纯 α 网络的加速比，是保守数字；② 它**不是逐帧延迟**。逐帧 demo
（batch=1）口径下，教师每帧要串行做 400 步优化，为**秒级**；学生一次小 MLP 前向，为**毫秒级**
——实时应用只有学生路径可行。两个口径都成立，但汇报时不混用。

## 5. 第五幕：天花板与剩余余量（强 oracle）

**oracle 是什么、为什么要跑**：把优化目标从"绳读数"换成 **GT 指尖位置**，其余（动作空间、
优化器、强配方）完全不变。这给出"该动作空间 + 该优化器"能达到的**上限**——余量 = oracle 增益
− rope 增益，即换更好的信号或方法后，在同一动作空间里**最多**还能挤出多少。它同时是防泄漏
证据：如果 5 个绳标量已经"泄漏"了答案，rope 就应贴着 oracle；实测存在稳定差距，且 5 标量
相对 63 维姿态答案（21 关节 × 3）信息量极小——不存在平凡泄漏。

| 评测设置 | oracle 增益（mm） | rope 冠军（mm） | 净余量（mm） | rope 达到 |
|---|---:|---:|---:|---:|
| FreiHAND mask70 | -1.97 | -1.68 | 0.29 | 85% |
| FreiHAND finger_end80 | -2.48 | -1.66 | 0.82 | 67% |
| HO3D v2 mask70（WiLoR） | -1.16 | -1.05 | **0.11** | **90%** |
| HO3D v2 mask70（HaMeR） | -1.29 | -0.66 | 0.63 | 51% |

**措辞纪律（重要）**：在 HO3D/WiLoR 上，绳修复已拿到 oracle 的 90%，净余量只有 0.11 mm——
**这条轴已接近该动作空间的天花板**，不能写成"后续改进空间很大"。任何图像条件化修复（P3 方向）
的论据只能建立在确有余量的轴上：finger_end80（0.82 mm）与 HaMeR（0.63 mm）。这个表同时划定了
方法论边界：若想突破 0.11 mm 这类天花板，出路不是更好的 α 预测器，而是更大的动作空间或更多
的信号维度——那是与本阶段不同的命题。

## 6. 必须主动交代的五个弱点（Must-state caveats）

以下五条在任何对外表述中主动交代，不等被问：

1. **模拟传感器**：绳读数由 GT 关节派生；第 3 节的噪声消融就是现实性边界。5 个标量携带的信息
   远少于 63 维姿态答案（第 5 节的 oracle 差距证明不存在平凡泄漏）。
2. **配方来源**：优化器超参（steps/lr/正则）是在评测 split 上选出的。缓解措施：该配方在 HO3D、
   finger_end80、HaMeR 上零重调迁移成立；学生侧超参未经调优，且 seed 稳定性 ±0.003 mm。
3. **协议边界**：所有增益均为同解码器口径；不得与早期硬基线表（0020，各 backend 原生解码协议）
   在关节级拼接比较（存在 ~0.3–0.5 mm 协议偏移）；跨表比较只用 mesh/F 类指标。
4. **干净图像**：本管线定性为 hard/遮挡样本修正器；clean split 上学生是中性（+0.02 mm），
   **不是**改进。
5. **有效教师多样性**：四个训练教师中的三个 FreiHAND 系教师共享同一批 32,560 张底图（区别只在
   扰动类型/backend）；数据集层面的多样性是 **2 个语料库**，不是 4 个。

## 7. 下一步：三条路线，请导师定夺

P2 已按预注册门槛闭环（教师 → 学生 → 发布模型 → 对照齐全）。往下三条路线互不排斥，但资源上
需要排优先级。我们把各自的论据、首步与风险摆出来，**不预设结论**：

### 路线 A：P3 v1——token/局部图像特征进学生

- **一行现状**：P3 v0 用 pooled 全局图像特征无增益且 image-only 对照归零（rope-only -1.636 mm、
  加 pooled 特征 -1.533 mm、纯图像 -0.100 mm），指向 token/局部特征（v1）。
- **论据**：图像条件化修复的空间只存在于有余量的轴——finger_end80（0.82 mm）与 HaMeR
  （0.63 mm）；HO3D/WiLoR 轴已近天花板（0.11 mm），对它不应有期待。v0 的 image-only 对照归零
  说明失败在表征（pooled 特征太粗、与指尖不对齐），不在训练管线——训练侧收敛正常。
- **首步**：特征缓存管线已支持 token 级导出（`--save-tokens`）；提取 token/grid 特征，做指尖
  投影邻域的局部池化或轻量注意力头，沿用全部对照协议（image-only、shuffle、clean 中性带）。
- **风险**：余量上限明确（<1 mm 级），属于"精细化"而非"开辟新空间"的工作。

### 路线 B：扩数据（首选 DexYCB）

- **论据**：全篇最弱的轴始终是跨数据集（恢复率 87–93%），且已有直接证据表明加训练数据能移动
  这条轴（HO3D v3 教师数据把恢复率从 87% 提到 93%，见 4.4）；弱点 5 说明有效数据多样性目前只有
  2 个语料库——扩数据正面回应这两点。DexYCB 是大规模真实手—物交互数据集，与遮挡场景的动机
  最贴合。
- **首步**：DexYCB 数据审计 → 生成 hard split 与绳标签 → 训练教师并**先过教师质量体检**
  （multi-v2 的纪律，见 4.5）→ 并入多教师学生。
- **风险**：管线工程量（新数据集适配）；multi-v2 已证明"加数据不等于变好"，教师质量体检是
  硬前提。

### 路线 C：真实绳传感器硬件 + 逐帧 demo

- **论据**：把弱点 1（模拟传感器）从"消融论证"升级为"实测证据"，是三条路线中唯一能改变主张
  性质的一条。规格已经由噪声曲线给定：测距精度 ~2.5 mm 内保 >80% 增益、5 mm 是可用边界（第 3
  节）；运行时方面，学生路径毫秒级/帧（4.6 的口径），实时 demo 可行。
- **首步**：传感器选型（拉线编码器/柔性拉伸传感等）→ 标定到归一化绳长口径 → 实测噪声谱对齐
  噪声曲线 → batch=1 逐帧 demo。
- **风险**：硬件工程与标定的时间成本；实测噪声可能超出 2.5 mm 规格（但即便如此，噪声曲线也
  能告诉我们离可用还差多少，不会白做）。

三条路线的性质不同：A 是模型精细化（收益上限已知且有限），B 是补最弱的泛化轴（有直接证据支持
但有工程量），C 是改变证据性质、也最接近应用叙事（有硬件不确定性）。请导师按下一阶段的目标
（论文完整性 / 泛化能力 / 系统展示）排优先级；A 与 B 在计算资源上可小规模并行探索。
